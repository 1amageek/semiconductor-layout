# N ラダー: 報告する CAD から、完成させる CAD へ

M1–M10（常時ライブ検証 + 単一編集ストリーム + 階層/配線/LVS）の上に載せる次期北極星。

> **すべての違反・未達意図は、計算済みの修復手段を伴う「実行可能なゴール」である。**
> 検証エンジンが知っていること（違反の幾何、合法位置、押し退け、意図との差分、電気特性）を
> 人への報告で終わらせず、ツール自身が完成まで運ぶ。人間と AI エージェントは同じゴール
> コマンド面を共有する。

ベンチマークは「Virtuoso の操作性」ではない。**正しさが機械検証され、修復が機械実行され、
意図が機械照合される編集体験**であり、既存商用 CAD はこれを持っていない。

## 依存関係

```
 N1 修復エンジン ──────► N3 finish-net ──► N5 ゴールコマンド面
 （違反→修復）            （意図→完成）       （人間=Agent 等価）
        ▲                     ▲
 N2 SDL（意図の一次入力化）────┘
 N4 電気層（R/C/τ/EM）── 全段に情報を供給
 N6 トラストダッシュボード ── 全段の verdict を常設表示
```

## N1: 修復エンジン — 違反は実行可能なゴール

- `LayoutRepair`: 違反 1 件に対する**計算済みの修復**（移動ベクトル / 寸法変更 / via パッド追加
  / ジャンパ etc.）。kind ごとの計算機:
  - minSpacing → DRD enforce 二分探索で求めた最小合法変位（shove 機構の流用）
  - minWidth / minArea → 不足量ぶんの成長（隣接違反を作らない方向を探索）
  - enclosure → ランディングパッド追加 or 既存メタル拡張
  - overlapShort → 切断（subtract）か再配線提案（N3 連携）
  - 計算不能なら **「修復不能 + 理由」を返す**（silent skip 禁止）
- UX: 違反行/マーカー選択 → 修復プレビュー（ゴースト）→ 1 キー適用（commitDelta 1 回 = undo 1 単位）。
  `Fix All` は収束ループ（適用→再検証→次）で、**振動検出 + 予算 + 全ロールバックの原子性**。
- 前提として P0 表示層（エラーバナー / 違反ズーム+巡回 / EIP 保留チップ / 停止理由 HUD）を吸収する
  — 修復可能なものを見せられないツールに修復は載らない。
- **オラクル**: 修復適用 ⇒ 対象違反が消え、違反総数が厳密に減少し、新たな error 級を生まない
  （満たせない修復は適用前に「不能」と判定されること）。Fix All ⇒ 不動点で停止し、残余は
  全件「不能理由」付き。

## N2: SDL — 回路意図を編集の一次入力に

- `.subckt`（M8 リーダ）を読み込むと:
  - 未配置デバイス一覧（kind/W/L/m）。ジェネレータ（MOSFETCellGenerator 系）でゴースト配置
    → クリック確定（placeInstance 経由）。
  - **意図フライライン**: 宣言ネットではなく **LVS unmatched 端子間**を結ぶ（reference の
    ネットトポロジが導く「まだ繋いでいない物」）。
  - **LVS 収束メータ**: matched/total デバイス・ネットを診断バーに常設。編集のたびに更新
    （live LVS は既設）。
- インポート GDS にも効くように「ラベル→ネット注釈」コマンドを含む（netID なしでは意図照合が
  空回りするため。N2 の入口整備）。
- **オラクル**: inverter/NAND の .subckt から、配置→配線を UI 動詞だけで進めると unmatched が
  単調減少し、完了時に LVS クリーン + DRC クリーンで終わる E2E テスト。

## N3: finish-net — 意図ネットの 1 コマンド完成

- 選択した意図ネット（N2 の unmatched ペア）を、autocompleter + shove + 層切替/via（M9 機構）で
  **1 コマンド完成**。検証ゲート: 完成後 DRC クリーン + 当該ネットの LVS match 反転が条件。
  満たせなければ**全ロールバック + 阻害理由**（窓不達 / shove 予算 / via 不能）。
- `finish-all-nets` は N1 Fix All と同じ収束規律。
- **オラクル**: 成功時 = LVS unmatched が正確に 1 減 & DRC クリーン。失敗時 = 文書 bit-exact
  不変 + 理由が機械可読。

## N4: 電気層 — 幾何ではなく回路を編集する

- tech にシート抵抗・単位容量（面積/フリンジ）・EM 電流密度限界を追加し、編集中のネットに
  **ライブ R/C/τ 推定**を表示（配線 tick ごと、HUD とインスペクタ）。
- **トラストアンカー**: 推定値は PEXEngine（実 Magic 抽出で検証済み）と trust fixture 上で
  許容誤差内一致を CI で固定（このプロジェクトの「外部オラクルと締結してから信ずる」流儀）。
- EM: ネットの電流仕様（意図側に付与）に対し幅不足を violation として既存チャネルに流す
  → N1 の修復（幅成長）が自動的に効く。
- **オラクル**: trust fixtures で editor 推定 vs PEXEngine 抽出の R/C 誤差 ≤ 取り決め値。

## N5: ゴールコマンド面 — 人間と Agent の等価性

- N1–N4 の全ゴール操作（fix(violation), fixAll(), finishNet(net), placeFromNetlist(),
  annotateNets()）を**決定的・検証付き・リプレイ可能なコマンド**として公開。キーマップと
  Agent API は同一実装を呼ぶ（単一編集ストリームの上位版 = 単一ゴールストリーム）。
- 実行ログは「コマンド + 前後の verdict」で記録され、そのまま再実行・監査できる。
- **オラクル**: 同一文書 + 同一コマンド列 ⇒ bit-exact 同一結果。UI 経由と API 経由の結果一致。

## N6: トラストダッシュボード — 全軸 verdict の常設化

- DRC / 接続性 / 制約 / LVS / 電気 / （EIP 保留・antenna stale 等の deferred 状態）を
  **1 枚のライブ評定**に集約: 各軸の verdict + staleness + 適用範囲（何が検証済みで何が
  対象外か）。BC4 evidence bundle のエディタ内ライブ版。
- 「クリーンに見える」ではなく「何がクリーンで何が未検証か」を常時言語化する。

## 横断規律（M ラダーから継承し追加）

- 修復・完成コマンドはすべて commitDelta 経由（undo 1 単位・全ライブ系追従）。
- 原子性: 部分適用を残さない。不能は不能と言う。
- 収束ループは予算 + 振動検出 + 不動点停止。残余は理由付き。
- 新しい「推定」（N4）は必ず外部オラクル（PEXEngine/Magic）と CI で締結してから表示する。
